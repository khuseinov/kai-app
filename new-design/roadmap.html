01b · пробелы

Где дыры

Что отсутствует в системе и почему это важно.

Уведомления — нет экрана
Kai может предупреждать о сроке визы, изменении рейса, ценовом дропе — но нет ни UI-паттерна, ни экрана для этих событий
P0 · Экраны
Документы / Passport Vault — нет
Паспортные данные, визы, страховки — Kai их знает (Memory), но нет отдельного UI для хранения и просмотра документов
P0 · Экраны
Пустое состояние чата — не задокументировано
room.html показывает живой чат, но не показывает экран до первого сообщения. Это первое что видит пользователь после онбординга
P0 · Экраны
Paywall / Подписка — нет экрана
settings.html показывает «Pro» бейдж, но экрана выбора плана и апгрейда нет. Нет описания того, что даёт Pro
P1 · Экраны
Skeleton / Loading states — нет
Нет паттернов для состояний загрузки контента (memory, trip detail). Shimmer-плейсхолдеры — обязательный компонент
P0 · Компоненты
Toast / Snackbar — нет
Нет стандартного паттерна для краткого системного фидбека (сохранено, скопировано, ошибка). Каждый экран решает это по-разному
P0 · Компоненты
dark.html покрывает только 4 из 15 экранов
voice, memory, onboarding, edge-states, landing в тёмной теме не задокументированы
P2 · Улучшения
Motion tokens не в JSON
Кривые анимации и длительности есть в CSS-комментариях, но не в design-tokens.json. Flutter не может их импортировать автоматически

02 · экраны

Новые экраны

8 экранов которых не хватает системе. Для каждого — 2–3 варианта реализации с разным подходом.

S-01 · Уведомления / Alert Center

Kai проактивно сигнализирует о важных событиях: истекает виза, изменился рейс, упала цена на отель. Сейчас нет ни паттерна, ни экрана для этих ситуаций.

Notification Center

Отдельный экран в Nav-панели рядом с Memory/Settings. Карточки-события с цветовой кодировкой, сгруппированные по поездке.

В в чат-ленте

System message в чате

Kai вставляет предупреждение прямо в поток разговора — как system bubble с иконкой. Не требует отдельного экрана.

Третий пункт в Nav между Memory и Settings. Иконка паспорта. Экран похож на memory.html но заточен под документы с датами истечения срока.

Первое что видит пользователь после онбординга. room.html этого не показывает. Критично для first-impression и активации.

Только tide curve и одна строка анимированного текста по центру — typewriter эффект перебирает разные вопросы. Минималистично, в духе zero-UI.

В абсолютный ноль

Чистый экран

Только tide curve, никакого контента. Compose island ждёт внизу. Экстремальный zero-UI — доверяем тому, что онбординг объяснил всё.

Как онбординг

3 экрана: что даёт Pro → выбор плана → оплата. Tide curve во всех состояниях. Кнопка «Не сейчас» всегда доступна.

В inline в Settings

Раскрывающаяся карточка

В settings.html карточка аккаунта раскрывается по тапу и показывает plan comparison. Нет отдельного экрана — всё на месте.

Shared Trip Folder

Multi-user Room

Отдельный тип чат-рума — «совместный». Сообщения других участников видны в ленте, Kai отвечает в контексте всего разговора. Имена участников над бабблами.

Посуточный план поездки — когда разговоры с Kai привели к конкретному плану действий. Нужен структурированный вид, а не только чат-история.

→ trip-detail.html

«Маршрут» — вторая вкладка

Trip Detail получает табы: «Чаты» | «Маршрут» | «Источники». В маршруте — хронологическая лента дней, каждый день раскрывается в список активностей.

Б карточки дней

Day-by-day карточки

Горизонтальный скролл заголовков дней вверху, ниже контент выбранного дня. Каждая активность — карточка с временем, описанием, иконкой типа.

P2Effort L

S-07 · Поиск / Discover Mode

Пользователь ещё не знает куда хочет — он исследует варианты. Нужен режим вдохновения/исследования отличный от целевого чата.

→ room.html · nav.html

Иконка поиска в Nav-панели. По нажатию — полноэкранный поиск с историей и предложениями Kai: «популярно сейчас», «на основе вашей истории».

Б чат-режим Discover

Специальный чат

Отдельный тип разговора — Discover Session. Tide curve в особом состоянии (голубой + пульс), Kai задаёт вопросы первым: «какие даты?», «один или с партнёром?».

P2Effort S

S-08 · Чеклист перед вылетом / Pre-trip

За N дней до поездки Kai формирует персональный чеклист: паспорт, виза, вакцины, деньги, страховка. Автоматически на основе данных пользователя.

→ trip-detail.html · memory.html

Kai проактивно отправляет чеклист в чат поездки за 7 дней до вылета. Чекбоксы прямо в карточке, состояние сохраняется. Нет отдельного экрана — всё в ленте.

Раздел «Готовность» в поездке

В Trip Detail появляется прогресс-бар «Готовность к поездке» + список пунктов. Автоматически отмечает выполненные на основе данных из Memory.

03 · компоненты

Новые компоненты

8 компонентов которых не хватает в components.html. От критичных системных (toast, skeleton) до богатых интерактивных (date picker, map card).

P0Effort S

C-01 · Toast / Snackbar

Краткое системное уведомление: сохранено, скопировано, ошибка отправки, memory fact added. Сейчас каждый экран обходится без этого паттерна.

→ components.html
А снизу над compose рекомендую

Floating pill

Pill-форм, появляется снизу над compose island. 3 варианта: нейтральный (ink-1), positive (зелёный), negative (красный). Auto-dismiss 3s, swipe to dismiss. Tide-pulse на «memory saved».

Б сверху под tide

Banner под tide curve

Slide-down из-под tide curve. Занимает всю ширину. Более заметен — хорош для важных уведомлений, плохо — перекрывает контент.

P0Effort S

C-02 · Skeleton / Loading States

Плейсхолдеры пока грузится контент Memory, Trip Detail, source cards. Сейчас ничего нет — пустые экраны или мгновенный рендер.

→ components.html · memory.html · trip-detail.html
А shimmer серый

Standard shimmer

Серые прямоугольники var(--surface-2) с анимацией gradient sweep слева направо. CSS-only, легко добавить на любой экран как класс .skeleton.

Б tide shimmer рекомендую

Branded tide shimmer

Тот же sweep но с tide-градиентом в 5% opacity. Уникально для Kai — загрузка ощущается как «Kai думает», а не просто «контент грузится».

P1Effort M

C-03 · Дата-пикер / Date Picker

Выбор дат поездки. Kai часто уточняет «когда планируете?» — нужен нативный компонент который появляется вместо клавиатуры.

→ components.html · room.html
А calendar grid в sheet рекомендую

Calendar bottom sheet

Kai присылает rich-компонент в чат — тап открывает calendar grid в шторке снизу. Диапазон дат, tide-градиент на выбранном, подтверждение одной кнопкой.

Б inline в чате

Date chip в ленте

Kai предлагает даты как интерактивные чипы прямо в бабble. Tap на чипе — выбран. Несколько вариантов рядом. Минималистично.

В NLP текст

Естественный ввод

«через 2 недели», «конец июня», «с 15 по 22» — Kai сам парсит. Подтверждение показывает конкретные даты. Нет calendar UI вообще.

P1Effort M

C-04 · Country / City Selector

Структурированный выбор направления поездки — с поиском, флагами, популярными вариантами. Нужен для создания поездки и для контекста в onboarding.

→ components.html · onboarding.html
А search sheet рекомендую

Поиск + популярные

Bottom sheet: поле поиска вверху, ниже — «Популярные направления» (8 стран с флагами), результаты поиска в реальном времени. Разделители по континентам.

Б inline в чате

Destination chips

Kai предлагает несколько направлений как интерактивные карточки прямо в ленте. Флаг + название + 1 строка контекста («безвизовый», «сезон: апрель–октябрь»).

P1Effort S

C-05 · Multi-select

Выбор нескольких элементов для массовых операций: удалить несколько фактов из Memory, выбрать несколько поездок для экспорта.

→ components.html · memory.html
А long-press mode рекомендую

iOS-паттерн выделения

Long-press на любом элементе → режим выделения: слева появляются круглые чекбоксы, снизу — toolbar с действиями (удалить, экспортировать). Эскейп или тап в пустое место — выход.

Б edit mode кнопка

Кнопка «Изменить»

Кнопка в appbar → режим редактирования. Более явный, хорош для пожилых пользователей или первого раза. Менее элегантный.

P1Effort S

Onboarding уже использует dots (хорошо), но нет стандартного компонента для других multi-step флоу: visa application, trip creation wizard.

Стандартизировать dots

Вынести паттерн из onboarding.html в components.html как canonical. Добавить: активный = tide gradient pill, completed = ink-1 dot, upcoming = ink-4 dot.

Горизонтальный степпер с номерами и лейблами — для сложных wizard-флоу (создание поездки, онбординг про). Активный шаг = tide gradient circle, пройденный = ink-2 с галкой.

Фрейм F-05: Streaming

Kai «печатает» — tide curve в состоянии «thinking» (анимированный pulse), в ленте появляется баббл Kai с cursor-blink. Текст появляется постепенно — typewriter эффект.

добавить

Фрейм F-06: Error bubble

Kai не смог ответить — баббл с мягким красным фоном (negative-wash), иконка, текст «не получилось найти ответ», кнопка «попробовать снова».

P1Effort S

I-03 · onboarding.html → Экран 5: Разрешения

Сейчас 4 экрана. Нет 5го — запрос разрешений (уведомления, микрофон). Это обязательный шаг до первого использования и критичный для voice mode.

→ onboarding.html
добавить рекомендую

Экран 5 · Разрешения

Два пункта с иконками: 🎤 Микрофон (для голосового режима) и 🔔 Уведомления (для алертов о визах и рейсах). Кнопка «Разрешить» → системный запрос iOS/Android. «Пропустить» доступна всегда.

альтернатива

Just-in-time запросы

Не добавлять отдельный экран — запрашивать разрешения в момент первого использования фичи. Лучше конвертирует, но сложнее в реализации.

P1Effort S

I-04 · voice.html → Sleep state + Wake

Сейчас 4 состояния голосового режима. Нет Sleep (экран тускнеет после молчания) и Wake (оживление по wake word). Это важно для ambient use case.

→ voice.html
добавить рекомендую

R5 · Sleep state

После 30с молчания — tide curve плавно затухает до 20% opacity, экран диммируется. Только пульс дыхания остаётся. Любой звук → Wake transition (tide curve оживает).

добавить

R6 · Wake word

Из Sleep: пользователь говорит «Kai» → tide curve мгновенно разгорается, переходит в listening state. Анимация перехода Sleep→Listening — главный момент оживления.

P2Effort L

I-05 · dark.html → Полное покрытие всех экранов

Сейчас тёмная тема задокументирована только для 4 экранов (room / nav / trip / settings). voice, memory, onboarding, edge-states, landing — не покрыты.

→ dark.html
расширить dark.html рекомендую

+7 экранов в dark.html

Добавить секции: voice dark (уже тёмный по правилу — задокументировать исключение), memory dark, onboarding dark (4 screen), edge-states dark, landing dark. Итого 11 экранов в одном файле.

data-theme на каждый файл

Toggle в каждом файле

Добавить кнопку светлый/тёмный в каждый HTML файл. Нет отдельного dark.html — тема переключается на месте. Мощнее для инженерного хендофа.
