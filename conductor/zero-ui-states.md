# План: Контроль состояний Zero UI и защита от конфликтов

## Цель
Реализовать строгий контроль состояний для нового Zero UI:
1. Исключить конфликт между голосовым вводом и текстовой клавиатурой (скрывать кнопку клавиатуры, когда Кай слушает).
2. Сделать интерфейс интуитивно понятным за счет динамического изменения текста на главном экране ("Слушаю вас...", "Анализирую...").
3. Добавить тактильный отклик (Haptic Feedback) при тапе по экрану для включения микрофона.

## Архитектура
1.  В компонент `ChatEmptyState` будет передаваться текущее состояние `KaiVoiceState`. В зависимости от него центральный текст будет плавно меняться через `AnimatedSwitcher` (например, с "Как я могу помочь вам?" на "Слушаю вас...").
2.  В `ChatScreen`:
    *   В обработчик `onTap` (активация голоса) будет добавлен вызов `HapticFeedback.lightImpact()`.
    *   Кнопка-клавиатура (FAB) будет обернута в `AnimatedScale`. Масштаб будет равен `1.0` в состоянии `idle` и `0.0` (полностью скрыта) во всех остальных состояниях (когда активирована волна).
    *   В пустой стейт `ChatEmptyState` будет пробрасываться вычисленный `voiceState`.

## Шаги реализации
- [ ] Шаг 1: Обновить `ChatEmptyState` для поддержки `KaiVoiceState` и динамического текста.
- [ ] Шаг 2: В `ChatScreen` добавить импорт `services.dart` для виброотклика, добавить `HapticFeedback`, передать состояние в `ChatEmptyState` и скрыть FAB при активности ИИ.
